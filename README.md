# WarehouseTsdSystem

Веб-приложение и REST API для автоматизации складского учета с использованием терминалов сбора данных. В рамках курсового проекта реализован модуль приемки товара: выбор документа поставки, сканирование штрихкода, ввод фактического количества, проверка товара, фиксация расхождений и обновление складских остатков.

## Возможности

- приемка товара по документу поставки;
- проверка товара по штрихкоду;
- ввод фактического количества;
- фиксация расхождений между документом и фактической поставкой;
- сохранение приходной операции;
- обновление складских остатков;
- REST API для взаимодействия с ТСД;
- автоматизированные тесты сервиса приемки.

## Технологии

- Java 21;
- Spring Boot 3;
- Spring Web;
- Spring Data JPA / Hibernate;
- H2 Database;
- Maven;
- JUnit 5.

## Быстрый старт

1. Установить JDK 21.
2. Установить Maven или использовать Maven из IntelliJ IDEA.
3. Скачать проект или клонировать репозиторий:

```bash
git clone https://github.com/Hryundel/WarehouseTsdSystem.git
cd WarehouseTsdSystem
```

4. Собрать проект:

```bash
mvn clean package
```

5. Запустить приложение:

```bash
mvn spring-boot:run
```

6. Проверить работу API:

```http
POST http://localhost:8080/api/receiving
Content-Type: application/json

{
  "documentNumber": "DOC-001",
  "barcode": "460000000001",
  "quantity": 10,
  "cellCode": "A-01"
}
```

Пример успешного ответа:

```json
{
  "operationId": 1,
  "message": "Приемка товара успешно сохранена",
  "discrepancy": false
}
```

## Запуск тестов

```bash
mvn test
```

## Минимальный сценарий проверки

1. Запустить приложение.
2. Отправить POST-запрос на `/api/receiving` с корректным штрихкодом.
3. Убедиться, что операция приемки сохранена.
4. Отправить запрос с неизвестным штрихкодом.
5. Убедиться, что система возвращает ошибку проверки.
